参考答案 记忆模块
短期记忆：对话上下文窗口，保留最近 N 轮对话历史，直接作为 prompt 输入。

长期记忆：向量数据库 + RAG 检索，将历史对话、知识库文档向量化存储，需要时通过语义检索召回相关片段。

核心取舍：短期记忆保证连贯性，长期记忆保证广度，二者配合使用。
痛点

刷面经这件事，
为什么总是效率很低？

我们都有过整理了一大堆面试笔记，却从来没真正看完过的经历。问题出在哪里？

📱

手机上看不了

面试笔记都存在本地电脑的 Markdown 文件里，通勤、排队、睡前这些碎片时间想复习，掏手机却打不开——文件在电脑上呢。

📖

逐篇阅读太低效

就算能打开，从头到尾读一遍也只是被动浏览。看着眼熟不代表真的会，缺少主动回忆的机制，记忆留存率极低。

核心功能

不止是卡片，
是一整套复习系统

从解析笔记到智能复习，每个环节都为高效记忆设计。

01

翻转抽认卡

正面显示题目，先自己思考答案；点击翻面看参考回答。平滑翻页动画，模拟真实抽认卡的触感，强化主动回忆。

02

智能间隔重复

标记「熟了」的题减少出现频率，标记「再来一遍」的题加权 ×3 多出现。把时间花在真正薄弱的地方，越刷越高效。

03

按公司/类型筛选

明天面美团？一键只刷美团的题。想练行为面试？切换到行为题分类。精准定位，针对性准备。

04

进度永不丢失

所有复习状态自动存入 localStorage，关掉页面再打开，熟度标记、进度计数全部保留。想什么时候刷就什么时候刷。

05

在线添加笔记

面试现场想到新题？路上灵光一闪？直接在应用里添加题目和答案，自动加入题库。不必等到打开电脑编辑 Markdown。

技术题 · 蚂蚁 二面
RAG 架构中，为什么需要向量数据库？
👆 点击翻面
参考答案 RAG
语义检索：传统关键词匹配只能找到字面相同的内容，向量数据库能理解语义，找到意思相近但用词不同的文档。

长尾覆盖：用户的问题千变万化，不可能所有问法都提前写进关键词库里，向量检索能更好地覆盖长尾表达。
