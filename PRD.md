## Problem Statement

用户在求职面试过程中积累了多篇面试经验笔记（markdown 格式），存放在本地硬盘。用户希望在手机上的碎片时间（地铁、排队、睡前）快速复习这些面试题，但当前面临两个痛点：(1) 手机无法直接访问本地 markdown 文件；(2) 即使能访问，逐篇阅读效率太低，缺少抽认卡式的主动回忆机制。

## Solution

构建一个 **PWA 抽认卡应用**：一次性的预处理脚本将 markdown 面经解析为结构化 JSON，PWA 加载该 JSON，以抽认卡模式呈现面试题——正面出题，点击翻面看答案，用户标记"熟 / 不熟"后自动调节题目出现频率。支持按公司和题目类型筛选。复习进度通过 localStorage 持久化，无需服务器。

## User Stories

1. 作为求职者，我希望在地铁上打开手机就能随机刷面试题，这样就能利用碎片时间积累知识
2. 作为求职者，我希望看到题目时先自己思考，再翻面看答案，这样能强化主动回忆
3. 作为求职者，我希望标记"熟了"的题目少出现、标记"不熟"的多出现，这样能把时间花在薄弱点上
4. 作为求职者，我希望在明天某家公司面试前只刷该公司的相关题目，以便针对性准备
5. 作为求职者，我希望区分技术题和行为面试题，以便在不同场景下选择刷题内容
6. 作为求职者，我希望看到一个简单的进度计数（已过 X 题 · 熟 Y），以便了解自己的复习状态
7. 作为求职者，我希望复习进度不会因为关掉页面而丢失，下次打开继续
8. 作为求职者，我希望这个应用能添加到手机主屏幕，打开时全屏、离线可用，像一个真正的 App
9. 作为面经维护者，我希望以后添加新的面试笔记后只需重跑一次预处理脚本，就能更新题库

## Implementation Decisions

**Platform: PWA (Progressive Web App)**

- 静态单页应用，零后端
- Service Worker 实现离线缓存
- Web App Manifest 实现"添加到主屏幕"和全屏体验
- 所有数据打包为 JSON 内置于应用中

**Seams (模块边界及测试点)**

1. 预处理脚本 (`scripts/parse_notes.py`) — 读取 markdown 源文件，输出 `cards.json`。在此 seam 测试：JSON schema 合法、所有 9 个源文件均被覆盖、Q&A 对提取完整
2. 卡片数据层 (`data/cards.json`) — 静态 JSON，PWA 启动时加载。在此 seam 测试：schema 校验、必填字段齐全
3. PWA 外壳 — Service Worker 注册 + manifest。在此 seam 测试：离线加载、添加到主屏幕行为
4. 筛选模块 — 基于公司和题目类型的标签选择器，返回过滤后的卡片数组。在此 seam 测试：单选/多选/全选逻辑
5. 抽认卡引擎 — 核心状态机：正面展示 → 翻面 → 标记熟/不熟 → 下一题。在此 seam 测试：翻转动画触发、标记后频率调整、空池处理
6. 进度持久化 — localStorage 读写封装，按卡片 ID 存储熟/不熟标记及时间戳。在此 seam 测试：写入后刷新保留、新卡片默认状态

**Tech stack**

- 纯 HTML + CSS + vanilla JS（无框架依赖，PWA 足够简单）
- 预处理脚本使用 Python 3（用户环境已有）
- Service Worker 使用最小化手写实现

**Data schema (cards.json)**

```json
{
  "cards": [
    {
      "id": "meituan-r1-q3",
      "question": "Agent 的记忆模块你是怎么设计的？",
      "answer": "分为短期记忆（对话上下文窗口）和长期记忆（向量数据库 + RAG 检索）...",
      "company": "美团",
      "round": "一面",
      "type": "technical",
      "tags": ["RAG", "记忆模块", "Agent架构"]
    }
  ]
}
```

**Card interaction design**

- 正面：仅显示题目文本，无来源标签
- 翻转：点击卡片任意位置触发 CSS 3D flip 动画
- 背面：显示参考答案，底部两个按钮——「熟了」「再来一遍」
- 右上角显示计数器「已过 X 题 · 熟 Y」
- 标记后自动加载下一题，间隔 300ms

**Filter design**

- 首页顶部极简筛选栏，一行两个标签组
- 公司标签：美团、蚂蚁、腾讯、阿里、宇树、成都智信、全部
- 类型标签：技术题、行为题、全部
- 默认全选，不选纯随机
- 筛选后仅从匹配池中抽题

**Progress model**

- 每张卡片的复习状态存入 localStorage，key 为 `flashcard-progress-{cardId}`
- 状态枚举：`unknown`（未见过）、`known`（熟了）、`review`（需要再刷）
- 抽题算法：`review` 权重 ×3，`unknown` 权重 ×2，`known` 权重 ×1
- 计数器实时从 localStorage 汇总

## Testing Decisions

- 测试粒度：只测外部行为，不测实现细节
- 预处理脚本：在 9 个实际源文件上跑，验证输出 JSON 的 schema 和数量
- 筛选模块：单元测试，验证各组合的过滤结果
- 抽认卡引擎：单元测试状态转换和权重抽题
- 进度持久化：单元测试读写循环
- PWA 外壳：真机手动测试（安装、离线、全屏）
- 不使用测试框架，手写断言函数

## Out of Scope

- 微信小程序版本
- Telegram/微信 Bot
- 多设备同步
- 云端存储
- 用户登录
- 题目编辑功能（编辑直接在源 markdown 文件中进行）
- 自定义添加新题目的 UI
- 图片或富媒体题目
