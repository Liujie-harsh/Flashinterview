1. FlashInterview简介

- 是什么： 一款 PWA 渐进式网页应用，纯前端、零后端，装到手机主屏幕就是一个全屏 App。打开即用、断网可用、数据全在本地。
- 面向谁： 广大求职者，尤其是技术岗（Java 后端、AI Agent、前端、算法）的应届生和面试密集期的同学。
- 主要功能：
  1. 🎴 翻转抽认卡：正面出题，点击翻面看参考答案，强制"先思考再对照"，强化主动回忆。已自动解析 9 份真实面经笔记，共 180 道精选题目。
  2. 🧠 智能间隔重复：标记"熟了"的题减少出现频率（权重 ×1），"再来一遍"加权 ×3 多出现，"未见过"权重 ×2。把时间花在真正薄弱的题上，越刷越高效。
  3. 🏢 按公司/类型精准筛选：支持美团、蚂蚁、腾讯、阿里、宇树、成都智信 6 家公司筛选；技术题、行为题、系统设计、算法题 4 种类型分类。明天面美团？今晚只刷美团的题。
  4. 💾 进度永不丢失：所有熟度标记、复习进度自动存入 localStorage，关掉页面再打开全部保留。
  5. ➕ 在线添加笔记：面试现场想到新题？直接在应用里手动录入或批量导入 Markdown，自动加入题库。
  6. 📱 PWA 主屏安装：Chrome/Edge 一键安装到主屏幕，iOS Safari 提供手动引导，离线也能刷。
  📷 建议评审在手机上打开体验链接，感受真实的抽认卡翻转手感与 PWA 安装流程。

2. FlashInterview创作思路

灵感来源

我自己就是用户。求职期间积累了 9 份 Markdown 面经笔记，存在电脑硬盘里。每次面试前想复习，要么坐电脑前读到犯困，要么掏手机发现文件打不开。后来接触到认知科学里的 "主动回忆"（Active Recall） 理论——比起被动阅读，主动提取答案能让记忆留存率提升 3-5 倍。Anki 这类工具已经验证了这个方法，但录入成本把大多数人挡在门外。

想解决的问题

1. 手机看不了本地 markdown：面经笔记躺在电脑硬盘里，碎片时间想复习却无能为力。
2. 逐篇阅读效率太低：从头读到尾只是被动浏览，没有"先想答案再对照"的机制，看完以为会了，关掉就忘。
3. 不分轻重缓急：会的题和不会的题混在一起刷，时间浪费在已经掌握的内容上。
4. 没有按公司精准定位：明天面美团，今晚却还在刷蚂蚁的题，针对性差。
5. 隐私顾虑：面试笔记涉及个人项目细节、公司内部信息，上传云端不放心。

为什么做这个方向

- 痛点足够真实：这是我自己每周都在经历的痛苦，不是臆想出来的需求。
- 技术方案足够简洁：PWA + Python 预处理脚本，零后端、零运维，一个人就能完成。
- 差异化清晰：相比 Notion 移动端（依赖网络、被动阅读）、Anki（录入门槛高）、刷题 App（题目固定无法用自己的面经），FlashInterview 的差异化是 "零成本导入 + 隐私本地化 + 主动回忆" 三位一体。
- 场景天然契合 PWA：碎片时间意味着网络不稳定（地铁、电梯），离线可用是刚需，PWA 正好满足。

4. TRAE 实践过程

4.1 完整开发流程

整个 Demo 从 0 到 1 完全使用 TRAE 完成，开发流程分为五个阶段：

  阶段                 	内容                                      	产出                                      
  Phase 0：方案打磨       	与 TRAE 进行头脑风暴，让其扮演技术官对方案进行盘问，输出 PRD 文档  	PRD.md                                  
  Phase 1：数据预处理      	用 Python 解析 9 份 markdown 面经，支持 5 种 Q&A 格式变体，输出结构化 JSON	scripts/parse_notes.py + data/cards.json（180 道题）
  Phase 2：PWA 核心功能   	抽认卡翻转、智能间隔重复、按公司/类型筛选、localStorage 进度持久化、在线添加题目	app/index.html + app/style.css + app/app.js
  Phase 3：PWA 离线 + 安装	Service Worker 缓存策略、Web App Manifest、图标生成、iOS 安装引导	app/sw.js + app/manifest.json + app/icons/*
  Phase 4：部署上线       	GitHub Pages 部署、404 与旧 Service Worker 劫持问题排查、缓存版本 bump 到 v2	线上可访问                                   

4.2 关键技术决策（与 TRAE 协作产出）

- 翻转动画选 2D crossfade 而非 3D transform：3D transform 在移动端会触发额外的合成层，部分低端机型有抖动和闪烁，最终采用 opacity + translateX 的 2D crossfade，视觉接近、性能更稳。
- PWA iOS 安装兜底：iOS Safari 不支持 beforeinstallprompt，TRAE 给出了"UA 检测 + 显示手动安装提示条（分享 → 添加到主屏幕）"的方案，并写好了关闭交互。
- Service Worker 缓存版本手动 bump：每次发布需手动升级 flashinterview-v2 → v3，并在 activate 阶段清理旧缓存，避免用户卡在旧版本。
- 数据源从 cards-data.js 迁移到 cards.json：早期用 JS 文
