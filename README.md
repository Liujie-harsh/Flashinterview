0. 作者自述

👋 我是一位大四学生，正在求职 AI 应用开发方向。每次面试前想复习面经，对着电脑坐不住，地铁上却什么都看不了。Anki 之类的抽认卡工具其实早已验证了"主动回忆"对记忆的巨大提升，但门槛太高：要手动一张张录入卡片、设置间隔重复参数。我只是想把已经写好的面经笔记自动转成抽认卡，把"录入"和"记忆"的成本降到零。

我把 TRAE 当成"既要陪我打磨方案、又要陪我落地代码"的全栈搭档

1. 头脑风暴阶段：先让 TRAE 扮演一个真实的技术官，对我抛出的方案进行盘问——"为什么不用小程序？""PWA 在 iOS 上的安装体验怎么解决？""localStorage 会不会被清空？"——通过这种"对抗式"对话，把脑子里的模糊想法逼成一个清晰的 PRD。
2. 方案落地阶段：基于讨论后的 PRD，让 TRAE 帮我拆分模块（预处理脚本、卡片数据层、PWA 外壳、筛选、抽认卡引擎、进度持久化），每个模块单独推进，避免一锅端。
3. 执行阶段：把每一步需求一句句讲给它听，比如"翻转动画用 2D crossfade 而不是 3D transform，因为移动端有合成层抖动问题""Service Worker 缓存版本要手动 bump 到 v2"——TRAE 帮我把这些决策直接落到代码里。
4. 踩坑复盘阶段：部署到 GitHub Pages 时遇到 404 和莫名跳转到第三方域名的问题，TRAE 陪我一步步排查，最后发现是浏览器里残留的旧 Service Worker 在劫持请求，清除后恢复正常。

最让我惊喜的坎：PWA 在 iOS Safari 上的安装体验一直是出了名的难搞（没有 beforeinstallprompt 事件）。我把这个问题丢给 TRAE 后，它直接给了我一套"检测 iOS → 显示手动安装提示条 + 分享按钮引导"的兜底方案，连提示文案和关闭交互都写好了。原本以为要折腾半天的兼容性问题，一次对话就解决了。

1. FlashInterview 简介

- 是什么： 一款 PWA 渐进式网页应用，纯前端、零后端，装到手机主屏幕就是一个全屏 App。打开即用、断网可用、数据全在本地。
- 面向谁： 广大求职者，尤其是技术岗（Java 后端、AI Agent、前端、算法）的应届生和面试密集期的同学。
- 核心价值： 把已写好的 Markdown 面经自动转成抽认卡，结合主动回忆 + 智能间隔重复，让碎片时间真正变成有效复习时间。

2. 创作思路

灵感来源

我自己就是用户。求职期间积累了 9 份 Markdown 面经笔记，存在电脑硬盘里。每次面试前想复习，要么坐电脑前读到犯困，要么掏手机发现文件打不开。后来接触到认知科学里的 "主动回忆"（Active Recall） 理论——比起被动阅读，主动提取答案能让记忆留存率提升 3-5 倍。Anki 这类工具已经验证了这个方法，但录入成本把大多数人挡在门外。

想解决的问题

1. 手机看不了本地 markdown：面经笔记躺在电脑硬盘里，碎片时间想复习却无能为力。
2. 逐篇阅读效率太低：从头读到尾只是被动浏览，没有"先想答案再对照"的机制，看完以为会了，关掉就忘。
3. 不分轻重缓急：会的题和不会的题混在一起刷，时间浪费在已经掌握的内容上。
4. 没有按公司精准定位：明天面美团，今晚却还在刷蚂蚁的题，针对性差。
5. 隐私顾虑：面试笔记涉及个人项目细节、公司内部信息，上传云端不放心。

为什么做这个方向

- 痛点足够真实：这是我自己每周都在经历的痛苦，不是臆想出来的需求。
- 技术方案足够简洁：PWA + Python 预处理脚本，零后端、零运维，一个人就能完成。
- 差异化清晰：相比 Notion 移动端（依赖网络、被动阅读）、Anki（录入门槛高）、刷题 App（题目固定无法用自己的面经），FlashInterview 的差异化是 "零成本导入 + 隐私本地化 + 主动回忆" 三位一体。
- 场景天然契合 PWA：碎片时间意味着网络不稳定（地铁、电梯），离线可用是刚需，PWA 正好满足。

5. 功能特性

🎴 1. 翻转抽认卡

正面出题，点击翻面看参考答案，强制"先思考再对照"，强化主动回忆。已自动解析 9 份真实面经笔记，共 180 道精选题目。

🧠 2. 智能间隔重复

标记"熟了"的题减少出现频率（权重 ×1），"再来一遍"加权 ×3 多出现，"未见过"权重 ×2。把时间花在真正薄弱的题上，越刷越高效。

  状态  	权重  	出现频率
  熟了  	×1  	最低  
  未见过 	×2  	中等  
  再来一遍	×3  	最高  

🏢 3. 按公司 / 类型精准筛选

- 公司：美团、蚂蚁、腾讯、阿里、宇树、成都智信 6 家 + 我的自定义
- 类型：技术题、行为题、系统设计、算法题 4 种分类
